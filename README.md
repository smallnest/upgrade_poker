# 升级（拖拉机）纸牌游戏

命令行版本的升级（拖拉机）扑克牌游戏，使用 Go 语言实现。

## 游戏简介

升级是国内非常盛行的4人扑克牌游戏，又称"80分"、"拖拉机"、"双升"等。本程序采用两副牌规则，2为常主。

## 规则概要

### 基本设置

- **人数**：4人，你（南方）与北方AI搭档，对抗西方AI与东方AI
- **牌数**：两副牌共108张，每人25张，8张底牌
- **分牌**：5=5分，10=10分，K=10分，总分200分

### 主牌

2为常主，主牌大小顺序：

```
大王 > 小王 > 主级牌(主花色) > 副级牌(其他花色) > 主2 > 副2 > 主花色A > ... > 主花色3
```

副牌大小顺序（同一花色内）：

```
A > K > Q > J > 9 > 8 > 7 > 6 > 5 > 4 > 3
```

### 亮主与反主

- 发牌过程中可亮主，优先级从低到高：
  1. **对级牌** — 两张同级牌，确定该花色为主
  2. **三张级牌** — 覆盖对级牌
  3. **对王（无主）** — 最高优先级，打无主
- 无人亮牌则打无主
- 亮主成功者成为庄家

### 出牌规则

- **跟牌**：必须跟首家出的花色，出牌数量与首家相同
- **对牌**：首家出对时，有对必出对
- **拖拉机**：相邻同花色联对（如KKQQ），出拖拉机时有拖拉机必跟
- **甩牌**：领出时可同花色多张联出，需保证每张/每对都是最大
- **毙牌**：无首家花色时可出主牌毙，拖拉机和对牌数目不得少于首家
- **盖毙**：多家毙牌时以拖拉机和对牌大小比较

### 升级记分

| 闲家得分 | 结果 |
|---------|------|
| 0分 | 庄家连升3级（大光）|
| 1-39分 | 庄家连升2级 |
| 40-79分 | 庄家升1级 |
| 80-119分 | 闲家上台（换庄）|
| 120-159分 | 闲家上台并升1级 |
| 160-199分 | 闲家上台并连升2级 |
| 200分以上 | 每多40分闲家再升1级 |

### 扣底与抠底

- 庄家取走8张底牌后扣8张牌
- 最后一轮若闲家赢，底牌分翻倍计入闲家得分
- 翻倍数由赢墩牌型决定

## 编译与运行

### 前置条件

- Go 1.26+

### 编译

```bash
make build
```

或

```bash
go build -o upgrade .
```

### 运行

```bash
make run
```

或

```bash
./upgrade
```

### 测试

```bash
make test
```

### 压力测试（50轮AI对战）

```bash
make stress
```

### 清理

```bash
make clean
```

## 操作说明

- 出牌时输入牌的序号，空格分隔
- 手牌按花色分行显示，主牌花色在第一行
- 跟牌时需出与首家相同数量的牌
- 庄家扣底时选择8张牌

### 示例

```
你的手牌（25张）：
  ♥(主): 1:大王 2:小王 3:10 4:10 5:2 6:2 7:A 8:K 9:Q 10:J 11:9 12:8 13:7 14:6 15:5 16:4 17:3
  ♠: 18:A 19:K 20:Q 21:J 22:9 23:8 24:7 25:6 26:5 27:4 28:3
  ♦: 29:A 30:K 31:Q 32:J 33:9 34:8 35:7 36:6 37:5 38:4 39:3
  ♣: 40:A 41:K 42:Q 43:J 44:9 45:8 46:7 47:6 48:5 49:4 50:3

你领出，请选择要出的牌（输入序号，空格分隔）：
18
```

## 项目结构

```
├── main.go        # 入口
├── card.go        # 牌的类型定义、创建牌组、排序、显示
├── rules.go       # 主牌判定、大小比较、拖拉机检测、出牌合法性校验
├── game.go        # 游戏状态、发牌、计分、升级、主循环
├── player.go      # 玩家类型、手牌管理
├── bid.go         # 亮主/反主逻辑
├── trick.go       # 一轮出牌（墩）的逻辑、赢家判定
├── ai.go          # AI出牌策略
├── display.go     # 终端显示辅助函数
├── game_test.go   # 单元测试与AI压力测试
├── go.mod         # Go模块定义
└── Makefile       # 构建脚本
```
